# Inbox 条目处理决策树 v1

对每条 Inbox，按顺序问：

1. 是否是明确可执行的动作？

   - 是 → 生成 Task，归入对应 Plan
   - 否 → 进入问题 2
2. 是否指向系统/项目结构调整？

   - 是 → 修改 Step / Hypothesis / Plan 结构
   - 否 → 进入问题 3
3. 是否是长期原则 / 约束 / 价值判断？

   - 是 → 写入 Plan 的 constraints / philosophy / rules
   - 否 → 进入问题 4
4. 是否只是情绪 / 提醒 / 模糊想法？

   - 是 → 要么提炼为“可操作规则”，要么合并 / 丢弃
